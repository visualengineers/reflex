Introduction

With this web application you can configure, debug and start different types of depth sensors for use with Elastic Display Client applications.

The Backend Framework is analyzing the depth image and reconstructs touch points (Single Touch or Multi Touch). Also, the Backend can be configured for broadcasting these touch events via TCP or WebSockets

Start Instructions

  1. Select the appropriate Depth Sensor and Resolution from Tracking Tab.

  2. Activate Live Preview to view Point Cloud received from depth sensor.

    in Point Cloud Visualization you can check:

    • Filtered Depth Points (should be red)
    • Min / Max Depth Distance Planes (red / blue planes)
    • Dimension/Extend of the complete point cloud
    • Display the current depth point position values on hover (selected depth point should turn dark blue)
    • If Processing is active, minimum and maximum values should be displayed as yellow points
  3. adjust settings to filter invalid areas and find the right values for Smoothing
  4. Activate Interaction Processing to compute depth touch points.
  5. For calibration of viewport and mapping of depth camera coordinates start interactive Calibration
    • Before Calibrating, activate interaction processing
    • Press Calibrate to switch to Calibration mode
    • If not already displayed in fullscreen, now hit F11 to enter fullscreen mode
    • Drag borders so the viewport fits to the projected area
    • Save border by Pressing Save Area
    • Turn on Update of interactions
    • Now select the point to calibrate (Point 1, Point 2 or Point 3
    • Push into the surface - Usually, only one extremum is displayed,a nd auto-selected. Click Set point to map this depth value to the calibration point. Do this for all 3 Calibration Points.
    • If more than one extremum is displayed, pause auto-update and select the extremum-value, you find most appropriate
    • After Setting all3 Points, click on Apply to compute Calibration Matrix
    • To validate you calibration, activate the Calibrated toggle. the displayed touch points should now be displayed in the correct positions.
  6. For broadcasting touch points to other applications, go to the Network Tab and select the appropriate Network Interface and configuration.

Troubleshooting / Known Issues

  • Please check the Log if something is not working as expected.
  • Some errors also may be printed to the Browser Console. Hit F12 to open it.
  • Web app (at least Calibration) should be executed in FullScreen (F11).

For Developers

Technologies

  • ASP.NET Core and C# for cross-platform server-side code
  • Angular and TypeScript for client-side code

Features

  • Client-side navigation.
  • Angular CLI integration. In development mode, there's no need to run ng serve. It runs in the background automatically, so your client-side resources are dynamically built on demand and the page refreshes when you modify any file.
  • Efficient production builds. In production mode, development-time features are disabled, and your dotnet publish configuration automatically invokes ng build to produce minified, ahead-of-time compiled JavaScript files.

The ClientApp subdirectory is a standard Angular CLI application. If you open a command prompt in that directory, you can run any ng command (e.g., ng test), or use npm to install extra packages into it.
